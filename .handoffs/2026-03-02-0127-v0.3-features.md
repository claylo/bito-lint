# Handoff: v0.3 Features — Config Discovery, Suppression Filtering, Custom Content

## Where things stand

**Everything is done and merged to main.** Three features shipped in a single commit on `feat/v0.3-features`, now merged to `main`. 282/282 tests pass, clippy clean, fmt clean.

### 1. Broader Config Discovery

- `find_project_config` replaced with `find_project_configs` returning `Vec<Utf8PathBuf>`
- `.bito.<ext>` and `bito.<ext>` names now discovered alongside `bito-lint` names
- All found files from the closest matching directory are merged via figment (last wins)
- Precedence (low to high): `.bito`, `bito`, `.bito-lint`, `bito-lint`
- `ConfigSources.project_file` changed to `project_files: Vec<Utf8PathBuf>`

### 2. Region-Level Suppression Filtering

- `disable`/`enable` directive blocks now filter individual findings post-analysis
- Affected checks: grammar issues, passive voice, sticky sentences, long sentences, complex paragraphs, echoes
- New `build_sentence_line_map()` and `build_paragraph_line_map()` in `text.rs`
- `strip_directives_preserving_lines()` blanks directive lines before building maps
- Aggregate counts recomputed after filtering
- Aggregate-only checks (readability, pacing, transitions, style) remain file-level only

### 3. Custom Content Entries

- `CustomEntry` struct with `instructions` (inline) and `file` (path) fields, `resolve()` method
- `custom: Option<HashMap<String, CustomEntry>>` on `Config`
- `bito-lint custom list` / `bito-lint custom show <name>` CLI subcommand
- `get_custom` MCP tool with `name` param
- `config_dir` field added to `ProjectServer` for file resolution

## Decisions made

- **Region-level, not finding-level suppression**: Suppression maps sentence/paragraph numbers back to source lines, then filters findings whose line falls in a suppressed region. We do not re-run analysis on sub-regions.
- **Config merge, not first-match**: When both `.bito.yaml` and `.bito-lint.yaml` exist, both load and merge via figment. `bito-lint` values override `bito` values.
- **Custom entries are name to markdown**: No persona-specific fields. Just name, inline instructions or file path. Plugins decide what names to look for.
- **`custom` subcommand, not `persona`**: The config section and CLI are generic (`custom`) since users may define voice guides, glossaries, style rules — not just personas.
- **MCP token budget raised to 4500**: Adding `get_custom` tool pushed past the 4000 limit.
- **`semi_sticky_count` preserved, not recomputed**: The sticky_sentences vec only stores >45% entries, so 35-45% entries cannot be filtered. Original count is kept.

## Landmines

- **`anyhow` is CLI-only**: Core crate uses `thiserror`. Don't use `anyhow` in `bito-lint-core`.
- **Directive lines confuse sentence splitter**: `<!-- bito-lint disable grammar -->` contains `!` which the sentence splitter treats as a boundary. That's why `strip_directives_preserving_lines()` exists.
- **`load_or_error()` double-traverses**: Calls `find_project_configs()` to check emptiness, then `load()` calls it again. Minor perf concern, not blocking.
- **Always use `cargo nextest run`**: Never `cargo test`. Clay's preferences explicitly require this.
- **Clay runs his own commits**: Uses `commit.txt` and conventional commit style. Don't commit unless asked.
- **Rust 2024 edition**: Explicit `ref` in pattern matches on references is a compiler error.
- **Subagents ignore test runner instructions**: Even with explicit "use nextest" directives, subagents sometimes fall back to `cargo test`. Reinforce in every prompt.
